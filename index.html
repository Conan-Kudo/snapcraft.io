Debian instructions

An official Debian package is available from the Debian Unstable archive. To install this:

$ sudo apt-get install snapd

Centos instructions

# install on Centos
sudo add-yum-repo https://rpm.snapcraft.io/path/
sudo yum install snapd

Fedora instructions

From a Fedora 23/24 machine, type:

$ wget
http://www.snapcraft.io/downloads/snap-confine-2.0-1.fc24.x86_64.rpm
$ wget http://www.snapcraft.io/downloads/snapd-2.0.8-1.fc24.x86_64.rpm
$ dnf install squashfs-tools
$ sudo rpm -Uvh snap-confine-2.0-1.fc24.x86_64.rpm snapd-2.0.8-1.fc24.x86_64.rpm

Red hat instructions

# install on Red hat
sudo add-yum-repo https://rpm.snapcraft.io/path/
sudo yum install snapd

A “snap” is a universal linux app

A snap is a Linux package that works on any distribution or device. Snaps are faster to install, easier to create, safer to run, and they update automatically and transactionally so your app is always fresh and never broken. Use snaps to get the newest version of any Linux app on your preferred Linux OS.

The public collection of snaps includes the latest and best apps from github and beyond, so you have the whole world of Linux apps at your fingertips. Take the tour below to experience “hello world” as a snap, or jump to the developer guide [link to http://snapcraft.io/create] to create your own snaps.

The “hello” world tour

This tutorial assumes Ubuntu 16.04 LTS on your desktop, cloud, VM or server. You may need to sudo apt install snapd before continuing.

Log in to the global snap store

The snap store holds a public collection of snaps, and delivers them to clouds, devices and private infrastructure. Sign in once from your device (you can require two-factor authentication for extra security) then you will be able to fetch snaps on demand:

Snap discovery and the global snap store

You can search the snap name or description:

$ snap find hello
Name           Version  Developer  Notes  Summary
hello          2.10     canonical  -      Make music using JFugue MusicStrings
hello-huge     1.0      noise      -      Hello world example
hello-world    6.1      canonical  -      Ubuntu Clock for the Unity 7 desktop

Install the snap you want with a simple command

sudo snap install hello

Run it:

$ hello
Hello world!

Snaps can also install services that run in the background, such as web servers or content management systems. Those will start automatically when you install the snap.

See installed snaps:

$ snap list
Name           Version               Rev  Developer
hello          2.10                  26   canonical
ubuntu-core    16.04+20160419.20-55  109  canonical
webdm          0.17                  21   canonical

Release channels — stable, candidate, beta and edge

You can choose if you want to get development versions of an app, by selecting the preferred “channel”. By default, snaps are installed from the stable channel, but there is also a candidate channel which is “soon to be the stable release”, a beta channel and an edge channel.

Choose how much risk you want to take on this app and set the channel accordingly with --candidate, --beta or --edge when you install the snap.

Switch from stable to the beta channel to get the latest development release:

snap install hello-world --beta

Know where your software comes from — snap publisher and details

The info command shows which channels are available:

snap info hello-world
XXX

Always fresh — update fast and reliably

Get the latest version of all your snaps by running:

snap refresh

Creating snaps is easy!

Snaps have a very simple internal structure — you can easily craft them by hand! But the easiest way to build a snap is with snapcraft, which supports building from source and from existing packages. Snapcraft also handles publishing your snaps to the world. Read how to create a snap and join the snap crafting community — we have a fun crowd of people who hang out on the #snappy channel on Freenode or on snapcraft@lists.snapcraft.io.
